# SocialZH

SocialZH — это мобильное веб-приложение на Django, представляющее социальную сеть с личными профилями, лентой постов, сообщениями, уведомлениями, системой ролей и дополнительными сервисами вроде викторины, генератора паролей, проверки пароля и сортировки слов.

---

## Обзор проекта

- Стек: Python, Django, SQLite, HTML/CSS, JavaScript.
- Архитектура: одно Django-приложение `accounts` внутри проекта `SocialZH`.
- Интерфейс: мобильная вёрстка с каруселью рекомендованных карточек, нижней таб-бар-навигацией и карточной лентой.
- Хранение файлов: `media/` для загруженных изображений, `static/` для стилевого и JS-кода.
- Хранение диалогов бота: `bot_data/`.

---

## Быстрый старт

1. Перейти в папку приложения:
   ```bash
   cd c:\Users\keepe\Desktop\data_analyz\SocialZH
   ```
2. Создать виртуальное окружение и установить Django:
   ```bash
   python -m venv env
   env\Scripts\activate
   pip install django
   ```
3. Применить миграции:
   ```bash
   python manage.py migrate
   ```
4. Запустить сервер:
   ```bash
   python manage.py runserver
   ```

> В `settings.py` включён `DEBUG=True`, используется SQLite-файл `db.sqlite3`.

## Структура и назначение файлов

### Корень проекта

- `manage.py`
  - Стандартный Django-скрипт для запуска сервера, миграций и администратора.

- `db.sqlite3`
  - Локальная база данных SQLite. Хранит пользователей, посты, чаты, уведомления и другую информацию.

- `tmp_test.py`
  - Вспомогательный тестовый скрипт на Django `Client` для проверки логина и отправки сообщений.

- `media/`
  - Папка для загруженных файлов: `posts/`, `profile_pics/`.

- `bot_data/`
  - Примеры JSON-историй бота: `bot_history_123.json` и другие.

### Проект Django: `SocialZH/`

- `SocialZH/__init__.py`
  - Отмечает пакет Django-проекта.

- `SocialZH/asgi.py`
  - ASGI-конфигурация приложения.

- `SocialZH/wsgi.py`
  - WSGI-конфигурация для хостинга.

- `SocialZH/settings.py`
  - Основные настройки Django:
    - `DEBUG=True`
    - `SQLite` как база данных
    - `STATIC_URL` и `MEDIA_URL`
    - `AUTH_USER_MODEL = 'accounts.CustomUser'`
    - `INSTALLED_APPS` включает только `accounts` и стандартные Django-приложения.

- `SocialZH/urls.py`
  - Корневые маршруты:
    - `admin/` → админка Django
    - `` → маршруты из `accounts.urls`
    - обслуживание `MEDIA` и `STATIC` в режиме разработки.

### Приложение `accounts/`

#### Модели: `accounts/models.py`

- `CustomUser`
  - Расширяет `AbstractUser`.
  - Дополнительные поля:
    - `role` (`user`, `moderator`, `admin`)
    - `profile_picture`
    - `coins`, `quiz_attempts`, `last_quiz_date`
    - `gold_border_until`, `verified_until`
    - `reg_stats`
    - `followers` (ManyToMany)
  - Свойства и методы:
    - `is_admin`, `is_moderator`, `is_regular_user`
    - `has_gold_border`, `has_verified`
    - `followers_count`, `following_count`
    - `check_quiz_attempts()` — логика ежедневных попыток викторины.

- `Hashtag`
  - Хранит теги.

- `Post`
  - Посты пользователей с изображением, подписью, локацией, хэштегами и упоминаниями.

- `SavedPost`, `HiddenPost`
  - Сохранённые и скрытые посты для текущего пользователя.

- `PostReport`
  - Жалобы на посты.

- `Chat`, `Message`
  - Модели чата между пользователями и обмена сообщениями.

- `Notification`
  - Уведомления о сообщениях, подписках, упоминаниях и системных предупреждениях.

#### Формы: `accounts/forms.py`

- `RegisterForm`
  - Форма регистрации на основе `UserCreationForm`.
  - Поля: `username`, `first_name`, `last_name`, `email`, `password1`, `password2`.

- `ProfilePictureForm`
  - Форма загрузки аватарки пользователя.

- `PostForm`
  - Форма создания/редактирования поста.
  - Поля: `caption`, `image`.

#### Админка: `accounts/admin.py`

- `CustomUserAdmin`
  - Кастомная админка для пользователей.
  - Поддерживает массовые действия:
    - сбросить попытки викторины
    - начислить монеты
    - деактивировать пользователей

- `PostAdmin`
  - Превью фото, поиск по тексту, фильтрация, сортировка.

#### Маршруты: `accounts/urls.py`

Основные URL-пути:

- Регистрация и логин: `register/`, `logout/`
- Лента и профиль: `feed/`, `profile/`, `profile/<username>/`
- Сообщения: `messages/`, `messages/chat/<user_id>/`, `messages/send/<user_id>/`
- Поиск: `search/`
- Магазин/админка/отчёты: `shop/`, `reports/`, `admin-dashboard/`
- Посты: `add/`, `post/<id>/`, `post/<id>/save/`, `hide/`, `report/`, `edit/`, `delete/`
- Утилиты: `generate-password/`, `check-password/`, `sort-words/`, `quiz/`
- Админские действия: `change-role/`, `delete-user/`, `set-coins/`

#### Представления: `accounts/views.py`

Это главный контроллер приложения. В нём реализованы:

- Регистрация, вход и выход пользователей.
- Просмотр профиля и чужого профиля.
- Подписки/отписки и список подписчиков.
- Лента постов с системой скрытия, сохранения и жалоб.
- Создание, редактирование и удаление постов.
- Чат между пользователями и хранение сообщений.
- Уведомления и отметка как прочитанные.
- Панель администратора и управление ролями / монетами.
- Игровая механика:
  - магазин
  - покупка улучшений
  - викторина
  - покупка дополнительных попыток
- Утилиты:
  - генерация пароля
  - проверка надёжности пароля
  - сортировка слов
- Локальный бот:
  - `bot_data/` сохраняет историю
  - `generate_bot_response()` отвечает на простые запросы

> При проверке проекта была удалена дублирующаяся функция `save_post` в `accounts/views.py`.

#### Тесты: `accounts/tests.py`

- Общие тесты для страниц профиля и поиска.
- Проверяют:
  - доступ к своему профилю и чужому профилю;
  - работу поиска по хэштегам и именам пользователей;
  - корректность обработки пустого результата.

#### Пользовательские фильтры: `accounts/templatetags/custom_filters.py`

- `format_post`
  - преобразует `#тег` в ссылку поиска
  - преобразует `@юзернейм` в ссылку на профиль

## UI/UX и визуальная часть

### `accounts/templates/accounts/base.html`

- Базовый макет приложения.
- Содержит мобильный контейнер `mobile-container`.
- Хедер с логотипом и кнопкой возврата.
- Нижнюю таб-бар-навигацию для быстрой смены разделов.

### Основные шаблоны

- `feed.html`
  - отображает ленту постов и карусель рекомендаций.
  - карточки постов с действиями: сохранить, пожаловаться, скрыть, скопировать ссылку.

- `profile.html`
  - отображает профиль пользователя, его посты, фото и данные.

- `login.html`, `register.html`
  - страницы аутентификации.

- `messages.html`, `chat_detail.html`
  - интерфейс чата и списка сообщений.

- `notifications.html`, `notifications_page.html`
  - страница уведомлений пользователя.

- `add.html`, `edit_post.html`
  - формы для создания и редактирования постов.

- `shop.html`, `quiz.html`, `reports.html`
  - дополнительные разделы с логикой внутри `views.py`.

### Стиль: `static/style.css`

- Мобильный дизайн с контейнером шириной `375px`.
- Палитра: фиолетово-синие акценты, мягкие тени и закруглённые карточки.
- Нижняя навигация и плавные переходы.
- Карточный интерфейс для постов с hover-эффектами.
- Пользовательская карусель рекомендаций.

### Скрипты: `static/script.js`

- Рендерит карусель рекомендаций.
- Реализует авто-прокрутку и остановку при взаимодействии.
- Делегирует клики по кнопкам постов:
  - удаление
  - сохранение
  - жалоба
  - скрытие
  - копирование ссылки
- Работа с CSRF-токеном и toast-уведомлениями.
- Отслеживание количества непрочитанных уведомлений.

## Рекомендации

- `settings.py` содержит секретный ключ в открытом виде. Для продакшна его следует вынести в переменные окружения.
- `DEBUG=True` подходит только для разработки.
- `db.sqlite3` лучше не публиковать в репозитории, если это рабочая база.
- `bot_data/` и `media/` можно перенести в `.gitignore`, если это тестовые данные.
